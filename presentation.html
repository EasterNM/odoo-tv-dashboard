Feature 1

Sales TV

แสดงรายการ SO พร้อมออกบิล จัดกลุ่มตาม เส้นทางจัดส่ง — รีเฟรชอัตโนมัติทุก 10 วิ

สิ่งที่แสดง

  • จัดกลุ่มตามเส้นทาง: กรุงเทพ / สายใน / สายนอก
  • ชื่อลูกค้า · เลขบิล easy-acc · สถานะทำบิล
  • ไฮไลต์ SO ที่ยอด Pick ≠ Pack (สีเหลือง)
  • QR code ด้านล่าง → สแกนเปิด Mobile รับบิล
  • Dark theme เหมาะจอ TV ขนาดใหญ่

ประโยชน์

ทีมเซลล์เห็นภาพรวมทันที

ไม่ต้องเปิด Odoo ค้นทีละออเดอร์

QR → Mobile ต่อเนื่อง

เซลล์เปิด TV แล้วพนักงานสแกนรับบิลได้เลย

เห็นปัญหา Pick≠Pack ทันที

แจ้งเตือนก่อนบิล เพื่อแก้ไขก่อนส่งของ

Feature 2

Store TV

แสดงสถานะงานคลัง 5 คอลัมน์ แบบ Kanban Board บนจอ TV

🔵

PICK

งาน picking ที่ยังค้างอยู่ ยังไม่เสร็จ

🟢

PACK

งาน packing ที่ยังค้างอยู่ ยังไม่เสร็จ

🟣

DELIVERY

งานรอส่ง ยังไม่ออกรถ รวมคลังเคลม

⚠️

Pick ≠ Pack

SO ที่ยอดไม่ตรง — แจ้งเตือนอัตโนมัติ

คอลัมน์ "รวม SO" — Cross-column View

มองเห็น SO หนึ่งใบอยู่ในขั้นตอนไหนบ้าง พร้อม elapsed time นานแค่ไหนแล้ว (เขียว → เหลือง → แดง)

Feature 4 — PWA

Mobile รับบิล

พนักงาน Store รับบิลจากเซลล์ — บันทึกลายเซ็นและสร้างเอกสารใน Odoo ได้จากมือถือ

  • 1
    สแกน QR จากหน้า Sales TV หรือเปิด URL ตรง
  • 2
    ติ้กเลือก SO ที่ได้รับบิลจริง
  • 3
    กรอกชื่อ + เซ็นลายมือ บน canvas
  • 4
    กด "ยืนยัน" ระบบทำงานอัตโนมัติ
  • 5
    รับเลขที่เอกสาร IT2026/0001

สิ่งที่เกิดขึ้นใน Odoo

✅ สร้าง Invoice Transfer (IT2026/XXXX)
✅ บันทึกชื่อผู้รับ + ลายเซ็น + เวลา
✅ mark รับบิลแล้ว = True ทุก SO
✅ แนบลายเซ็น + post chatter แต่ละ SO
✅ เชื่อม SO ผ่าน One2many line

ติดตั้งเป็น PWA ได้เลย

ไม่ต้องโหลดจาก App Store · มีไอคอนบนหน้าจอหลัก · เปิด fullscreen

Feature 5 — PWA

Tablet ขึ้นรถ

พนักงานคลังยืนยันสินค้าขึ้นรถ — สร้างเอกสาร + PDF สรุปได้ในไม่กี่วินาที

  • 1
    เลือกเส้นทาง ปุ่มใหญ่แต่ละสาย + จำนวน SO
  • 2
    ดูตาราง SO จัดกลุ่มตามขนส่ง · ดู ชิ้น/แพ็ค/สถานะบิล
  • 3
    ติ้ก SO ที่ขึ้นรถแล้ว (หรือเลือกทั้งหมด)
  • 4
    กรอกรถ/คนขับ/เวลา
  • 5
    PDF download อัตโนมัติ + บันทึก Odoo

สิ่งที่เกิดขึ้น

✅ สร้าง Dispatch Transfer (DT2026/XXXX)
✅ link SO ผ่าน Many2many
✅ mark ขึ้นรถแล้ว = True ทุก SO
✅ สร้าง PDF ใบสรุป (A4 Landscape, Thai)
✅ แนบ PDF เข้า chatter ของ Dispatch record
✅ post chatter แต่ละ SO ระบุเลขเอกสาร

PDF ใบสรุปขึ้นรถ

มีข้อมูลรถ · ตาราง SO ทุกใบ · ยอดรวม · ช่องเซ็นชื่อคนขับ/ผู้รับ

Technical Deep Dive

Odoo Integration

ระบบดึงและเขียนข้อมูลตรงผ่าน XML-RPC — ไม่ต้องแก้ source code Odoo

Custom Models ที่สร้างใน Odoo Studio

Modelใช้สำหรับ
x_tv_dashboard_invoiceบันทึกรอบรับบิล IT2026/XXXX
x_tv_dashboard_invoice_lineรายการ SO แต่ละบิล
x_tv_dashboard_dispatcบันทึกรอบขึ้นรถ DT2026/XXXX

Custom Fields บน sale.order

Fieldความหมาย
เส้นทางการจัดส่งกรุงเทพ / สายใน / สายนอก ...
รับบิลแล้วBoolean + เวลารับบิล
ขึ้นรถแล้วBoolean
เลขบิล easy-accเลขอ้างอิงจากระบบ ERP เดิม

Thread-safe XML-RPC Client

odoo_client.py ใช้ thread-local proxy
รองรับ concurrent request จาก FastAPI
authenticate ครั้งเดียว cache uid ไว้

ไม่มี Database แยก

ดึงข้อมูลตรงจาก Odoo ทุกครั้ง ไม่มี sync lag · ข้อมูลถูกต้องเสมอ

Odoo 18 Quirks ที่แก้ได้แล้ว

attachment_ids ใช้ [att_id] · HTML sanitize ใช้ <p><ul> · many2many ใช้ [[6,0,ids]]
